## 📱 About TOK

TOK is a real-time chat application that lets users sign up, discover other people on the platform, send and manage friend requests, and chat one-on-one — with live message delivery, read receipts, online presence, and a personalized activity feed.

The name comes from Cameroonian Pidgin slang — "tok" (talk) — reflecting the app's core purpose: helping people talk to each other, simply and directly.

## ✨ Features

### Authentication
- Email & password sign up / login (single toggle screen)
- Friendly, human-readable error messages (no raw Firebase error codes)
- Persistent login sessions — users stay logged in across app restarts
- Forgot password flow via email reset link
- Display name set at signup, shown throughout the app

### Friend System
- Users must send a **friend request** before they can chat with someone
- Incoming requests can be **Accepted** or **Rejected** directly from the Discover screen
- Chat access is gated — even direct navigation to a chat screen is blocked unless the friendship is accepted
- Rejected requests are permanently marked as declined

### Messaging
- Real-time one-on-one chat, powered by Cloud Firestore listeners
- Message timestamps (HH:MM)
- Read receipts — single tick (sent) vs. double tick (read by recipient)
- Last-message preview shown in the friends list
- Unread message counts, shown as badges throughout the app

### Presence
- Live **online / offline** status, tracked via app lifecycle (foreground = online, backgrounded/closed = offline)
- "Last seen" timestamps shown when a user is offline

### Home Dashboard
- Real-time stats cards: total friends, total messages (sent + received), total unread messages
- **Recent Activity feed** — a live, merged timeline of:
    - New unread messages
    - Incoming friend requests
    - Friend requests that were accepted
- Activity items disappear automatically once "treated" (message read, request responded to, acceptance acknowledged) — no manual dismissal needed, no separate notification log to maintain

### Navigation
- Bottom navigation bar with 4 tabs: **Home**, **Discover Users**, **My Friends**, **Profile**
- Live badge counts on Discover (pending requests) and Friends (unread messages) tabs

### Profile
- View display name, email, and avatar (initials-based)
- **Light / Dark mode toggle**, switch instantly
- Sign out

### Branding
- Custom app icon and in-app logo — a coral chat-bubble mark, designed to work as both a launcher icon and an in-app brand element
- Consistent color system shared across light and dark themes

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| Framework | Flutter (Dart) |
| Authentication | Firebase Authentication (Email/Password) |
| Database | Cloud Firestore (real-time listeners) |
| State/Streams | `StreamBuilder`, `ValueNotifier`, `rxdart` (`CombineLatestStream`) |
| Presence tracking | `WidgetsBindingObserver` + Firestore |

> **Note:** Firebase Storage (for profile pictures and file/image attachments in chat) and Cloud Functions (for push notifications) require the Blaze (pay-as-you-go) billing plan and are not yet implemented in this version. See [Roadmap](#-roadmap) below.

## 🗄️ Firestore Data Model

### `users/{uid}`
```
email: string
uid: string
displayName: string
isOnline: boolean
lastSeen: timestamp
```

### `friend_requests/{pairKey}`
> `pairKey` = the two user UIDs, sorted alphabetically and joined with `_`
```
from: string (uid of requester)
to: string (uid of recipient)
participants: [uid, uid]
status: "pending" | "accepted" | "rejected"
acceptedSeenByRequester: boolean
timestamp: server timestamp
```

### `chats/{pairKey}`
```
participants: [uid, uid]
lastMessage: string
lastMessageTime: server timestamp
lastSenderId: string
```

### `chats/{pairKey}/messages/{messageId}`
```
senderId: string
participants: [uid, uid]
text: string
timestamp: server timestamp
read: boolean
```

## 🚀 Getting Started

### Prerequisites
- Flutter SDK installed
- A Firebase project (Firestore + Authentication enabled, Email/Password provider on)
- FlutterFire CLI configured (`firebase_options.dart` generated for your own Firebase project)

### Setup
```bash
flutter pub get
flutter run
```

### Firestore Indexes
Several screens use compound queries (`array-contains` combined with `where`). On first run, Firestore may prompt with a `FAILED_PRECONDITION` error containing a link to auto-create the required composite index. Click through any such prompts in the terminal the first time each query runs.

### Build a release APK
```bash
flutter build apk --release
```
Output: `build/app/outputs/flutter-apk/app-release.apk`

> ProGuard rules protecting Firebase classes are defined in `android/app/proguard-rules.pro` — required to prevent release-mode crashes caused by code shrinking stripping Firebase's reflection-based classes.
